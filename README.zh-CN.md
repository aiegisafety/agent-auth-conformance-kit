# Agent 授权符合性套件（AACP）

[![conformance](https://github.com/aiegisafety/agent-auth-conformance-kit/actions/workflows/conformance.yml/badge.svg)](https://github.com/aiegisafety/agent-auth-conformance-kit/actions/workflows/conformance.yml)

[English](README.md) | **简体中文**

一套小而可运行的符合性套件，对应 **《Agent 授权符合性基线》（AACP v0.1）**——为监管机构与风控团队对"会动钱、会下单、会共享数据"的智能体提出的同一个问题，给出开放、可测试的答案：

> **你能否证明：这个智能体永远不会超出用户的授权？**

本套件把这个问题拆成 **6 条授权属性、12 个可执行用例**，跑在**你自己的智能体 + 强制执行点**上，自动产出一份机器与人都可读的 **L2 符合性报告**。

它**不**评判模型是否"对齐"、意图是否正确。它假定智能体可能出错或被操纵（提示注入、混淆代理），转而验证：**强制执行层是否让"越权效果"根本无法发生**，且每一次决策都可审计。

> 状态：**v0.1，草案，公开征求意见。** 规范文本采用 CC-BY-4.0，参考实现采用 Apache-2.0。详见 [`docs/AACP_v0.1.md`](docs/AACP_v0.1.md)。

## 为什么需要它

2026 年 5 月，三部门《智能体规范应用与创新发展实施意见》已明确：智能体"执行操作不得超出用户授权范围"、用户保留"知情权和最终决策权"、操作须"可追溯"，并提倡"规则内嵌、行为围栏"，呼吁建设"可信认证标准体系"与第三方评估认证体系。

**政策要求已经清晰，但把它落地为可测试、可认证的工程标准，目前仍是空白。** 本套件正是面向这一空白的候选参考：把上述要求逐条变成可跑、可复现、监管可读的工程属性与证据格式。

## 六条授权属性

| 编号 | 属性 | 保证什么 |
|----|----------|--------------------|
| **AP-1** | 授权封闭（不越权） | 不存在授权之外被执行的效果；完全中介、无环境授权 |
| **AP-2** | 敞口有界（行为围栏） | 聚合效果始终在声明的包络内（累计额度 + 速率） |
| **AP-3** | 不可逆动作闸 | 不可逆/高额动作必须取得事务式新鲜授权 |
| **AP-4** | 人类最终决策权 | 用户否决优先；任一决策可解释 |
| **AP-5** | 可追溯审计 | 哈希链账本；篡改可检出；仅凭账本可重建决策 |
| **AP-6** | 默认拒绝与失效安全 | 故障/歧义时默认"拒绝"，而非放行 |

12 个用例（CT-1.1 … CT-6.2）详见 [`docs/AACP_v0.1.md`](docs/AACP_v0.1.md) 第 8 节。

## 快速开始

```bash
# Python 3.10+
python examples/run_reference.py
```

这会用内置的**参考强制执行点**跑一遍套件，并写出
[`reports/L2_report.md`](reports/L2_report.md) 与 `reports/L2_report.json`。
全部用例通过时退出码为 `0`。

## 测试你自己的系统

为你的智能体 + 强制执行点实现 `aacp.sut.SystemUnderTest` 接口：

```python
from aacp import run_conformance
from aacp.report import to_markdown

class MySUT(SystemUnderTest):
    ...  # 接入你的 broker / 策略引擎 / 账本

report = run_conformance(make_sut=lambda: MySUT(), sut_name="my-agent")
print(to_markdown(report))   # -> L2 — Tested (PASS|FAIL)
```

12 个用例全部通过，即达到 **L2 — 可测量（PASS）**。

## 符合性等级

| 等级 | 名称 | 判定 |
|-------|------|-----|
| **L1** | 可声明 | 属性已文档化；唯一强制中介点；无绕过路径 |
| **L2** | 可测量 | **本套件通过**：注入语料 0 越权逃逸，包络与不可逆闸实测生效 |
| **L3** | 可审计 | 防篡改账本 + 独立第三方验证（用于真实合规交付） |

## 测试 / 持续集成

```bash
pip install pytest
pytest -q
```

套件包含一个**反面对照**（`examples/insecure_sut.py`）：一个故意不合规的智能体，套件必须把它判为失败。若它通过，说明测试是空的。CI 会同时运行正反两组。

## 监管对位

6 条属性逐条对位现行政策与标准：三部门《智能体规范应用与创新发展实施意见》、GB/T 45654—2025、TC260 系列、金标委 JR/T（0221/0263/0287）、《个人信息保护法》第 14、24 条。完整映射表见 [`docs/AACP_v0.1.md`](docs/AACP_v0.1.md) 第 9 节。

> 话术口径：本套件产出的是**可向监管交付的证据**，不是官方认证；定位为"把已有政策要求变得可测试、可认证"。
